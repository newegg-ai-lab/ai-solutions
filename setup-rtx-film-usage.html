Using ComfyUI Day to Day

Once everything is installed, this is what a normal session actually looks like — as opposed to the one-time setup and validation.

Starting Up

Every Session
PowerShell 
cd <INSTALL_DIR>\ComfyUI
python main.py --listen 127.0.0.1 --port 8188

Leave that window open, then open http://127.0.0.1:8188 in a browser. This is the one manual step every session — ComfyUI doesn't auto-start as a background service by default.

The Basic UI

  • Center canvas: the node graph — boxes connected by lines representing the pipeline (load model → encode prompt → sample → decode → save).
  • Queue button (bottom right): runs the current graph.
  • Queue panel (left or via the menu): shows pending/running/completed jobs and thumbnails of finished outputs.
  • Nodes are added via right-click → Add Node, or double-click on empty canvas to search by name.

Starting a Workflow — Three Ways

Option 1 · Easiest
Built-in templates

ComfyUI ships with template workflows already installed on this machine — go to the menu (top-left icon) → Workflow → Browse Templates. There are ready-made SDXL, ControlNet, and video templates you can load directly without building anything from scratch.

Option 2
Reload a previous workflow

Every image/gif ComfyUI saves has the workflow embedded in it — drag any file from ComfyUI\output\ onto the canvas and it rebuilds the exact graph that made it. This is the fastest way to pick up where a previous session left off.

Option 3
Save your own as a starting point

Once you've built or tweaked a graph you like, Workflow → Save (or Ctrl+S) stores it as a .json in ComfyUI\user\default\workflows\ — from then on it shows up in the workflow browser alongside the built-in templates.

Normal Generation Loop

  1. Load or open a workflow.
  2. Click into the prompt text box (the CLIPTextEncode node) and type/edit what you want.
  3. Adjust seed/steps/resolution if needed (in the KSampler and EmptyLatentImage nodes).
  4. Click Queue Prompt.
  5. Watch the progress bar on the running node; when done, the result appears in the Queue panel and saves to ComfyUI\output\.
  6. Don't like it? Change the seed or prompt and queue again — this is the fast iteration loop, seconds per image on this GPU.

Using the Specific Tools Already Installed

  • ControlNet / AnimateDiff / upscaling nodes don't appear until you add them: right-click → Add Node → browse to categories like "ControlNet Preprocessors," "Animate Diff," or search by name (e.g. type "RIFE" or "Upscale"). The templates from the previous section already have these wired up correctly if you'd rather start from a working example than assemble one node-by-node.
  • Model dropdowns (checkpoint, ControlNet model, motion module) auto-populate from what's already downloaded on this machine — everything from setup shows up there.

Batching Multiple Shots

Queue Prompt can be clicked repeatedly to stack jobs — each queues behind the last and runs in order, so you can queue several prompt variations and walk away while the GPU works through them, then review the Queue panel's thumbnails afterward.

Shutting Down

⏹️
Ctrl+C in the PowerShell window running main.py, or just close that window — there's no persistent background service to separately stop.
