06 启动并配置 Telegram Bot

Step 6
启动服务
telegram-bot.service
bash 
systemctl start telegram-bot.service

配置环境变量

bash 
vim /usr/local/abs-ai/telegram-bot/.env
.env 
ALLOWED_USER_IDS=Your_ALLOWED_USER_IDS
TELEGRAM_BOT_TOKEN=Your_TELEGRAM_BOT_TOKEN
ONYX_BASE_URL=http://localhost:3000
ONYX_SERVICE_ACCOUNT_TOKEN=Your_ONYX_SERVICE_ACCOUNT_TOKEN
ONYX_PERSONA_ID=1
ONYX_TIMEOUT=180

如何获取 Telegram 配置参数

获取 TELEGRAM_BOT_TOKEN
通过 @BotFather 创建 Bot
  1. 打开 Telegram，搜索 @BotFather
  2. 发送 /newbot
  3. 输入 Bot 名称（如 My AI Bot）
  4. 输入 Bot 用户名，必须以 bot 结尾（如 myai_bot）
  5. BotFather 回复您的 Token，复制保存
获取 ALLOWED_USER_IDS
获取您的 Telegram 数字用户 ID
  1. 搜索 @userinfobot
  2. 发送 /start
  3. 机器人将回复您的数字 User ID，复制保存

07 使用示例

在 Telegram 中向 Bot 发送消息：

👤
ABS_AI 四个核心
🤖
根据 ABS AI 的相关资料，其"四个核心"是指构建企业私有 AI 解决方案的四大关键组件，缺一不可。具体如下：

四个核心组件

🧠
vllm.service
1. 本地 LLM
运行 Qwen 3.6 35B 模型，部署在 NVIDIA RTX PRO 6000 Blackwell 96GB 显卡上。采用 MoE 架构，推理时仅激活约 3B 参数。所有数据全程本地，无第三方留存，无 API 账单。
📚
Onyx RAG
2. Onyx 企业知识库
将公司 SOP、产品手册、内部文件注入系统。AI 基于"你的资料"回答（RAG 技术），确保准确性和相关性，适用于新人上手、查规范、找答案。
💾
mem0.service
3. Mem0 记忆层
AI 记住每次对话、用户偏好及项目脉络。无需每次重新交代背景，AI 越用越懂团队，提供个性化且连贯的交互体验。
💬
telegram-bot.service
4. Telegram Bot 接口
通过熟悉的 Telegram 聊天窗口与 AI 交互。无需学习新工具，支持随时随地（含移动端）发问，降低使用门槛。

为什么选择这套架构？

优势 说明
合规无虞 敏感数据（MNPI、客户资料、财务数字）不出机房，满足严格信息安全要求
零边际成本 一次硬件投资，后续无限次使用，不按 Token 计费
开箱即用 硬件、软件、模型、配置一体化交付，节省自建技术栈工程成本
可成长 模块化架构预留升级空间，可随团队规模扩展
